# 국제화 (i18n)

i18next + react-i18next 기반 국제화 시스템

## 파일 구조

```
i18n/
└── locales/
    ├── en/                # 영어
    │   ├── common.json
    │   ├── navigation.json
    │   ├── home.json
    │   └── {컴포넌트}.json
    └── ko/                # 한국어
        ├── common.json
        ├── navigation.json
        ├── home.json
        └── {컴포넌트}.json

src/
└── i18n/
    └── index.ts           # i18next 설정 및 리소스 등록
```

## 사용법

```tsx
import { useTranslation } from 'react-i18next';

function Component() {
  const { t } = useTranslation('컴포넌트명');

  return <h1>{t('title')}</h1>;
}
```

## 번역 파일 예시

```json
// i18n/locales/ko/button.json
{
  "title": "버튼",
  "description": "버튼 컴포넌트 사용법",
  "basicUsage": {
    "title": "기본 사용법",
    "description": "기본 버튼 예시입니다."
  }
}
```

## 규칙

1. **새 번역 추가 시**: en/ko 모두 업데이트
2. **키 구조**: 중첩 객체로 섹션 구분
   - 예: `basicUsage.title`, `props.mode`
3. **네이밍**: camelCase 사용

## 새 컴포넌트 추가 시

1. `i18n/locales/en/{컴포넌트}.json` 생성
2. `i18n/locales/ko/{컴포넌트}.json` 생성
3. `src/i18n/index.ts`에 import 추가 및 resources에 등록
4. 동일한 키 구조 유지

## i18next 설정

언어 설정은 localStorage의 `locale` 키로 저장되며, 기본값은 'ko'

```typescript
// src/i18n/index.ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'

i18n.use(initReactI18next).init({
  resources: { en: {...}, ko: {...} },
  lng: localStorage.getItem('locale') || 'ko',
  fallbackLng: 'ko',
})
```
